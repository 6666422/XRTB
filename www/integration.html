Integrating RTB4FREE With Your Current Ad Service Infrastructure

If you already are serving ads using a system like XXX or YYY, there are some integration issues before you can use RTB4FREE to provide RTB access. These issues involve campaign management, accounting, and data analysis. Each of these issues will be discussed in greater detail below

Overview

Data Analysis

A great deal of market intelligence flows into RTB4FREE when taking in bid requests; and yet more data is generated from your campaigns as they bid, win, and serve impressions. But you don't get all this 'big data' organized for you out of the box.

However, RTB4FREE provides you with publish/subscribe channels that log bid requests, bids, win notifications, pixel loads and click throughs. The data from these channels can be logged to Elastic Search, Hadoop's HDFS, Mongodb, flat files.

RTB4FREE comes standard with Elastic Search and flat-file loggers. Elastic Search is really exciting because this system comes out of the box with data analysis tools. All you need to do to use Elastic Search with RTB4FREE is to set up the Elastic Search system and use the Kibana reporting system with it. Here we show you how this is done in a more detailed section, below

Campaign Administration

Accounting

Accounting data in RTB4FREE comes from the win notifications sent by the RTB exchange. Also, if you configure RTB4FREE to handle your pixel loads and click throughs, then this accounting data is also available. All of this data comes in through the REDIS publish/subscribe channels.

Cost data is transmitted on the "wins" channel. A sample tool called "SqlRollUps.java in the src/tools directory can be used to record and rollup this accounting data into MySQL tables to maintain account, campaign and impression cost accounting. Using this tool, you can roll up the accounting data into your own accounting system. Or, you can use the tool as the starting point for your own rollup processing
